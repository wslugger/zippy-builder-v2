---
description: Start new features by creating a PLAN.md
---

1. Read `AGENTS.md` and `ARCHITECTURE.md` to understand the context.
2. Analyze the user's feature request.
3. Create a `PLAN.md` file in the root with:
   - Feature summary
   - Impacted files
   - Step-by-step implementation plan
   - Verification steps (unit and e2e)
4. Ask the user for feedback or approval before starting.
